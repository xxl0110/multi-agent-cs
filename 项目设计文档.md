## 一、项目定位

### 与第一个项目的对比

| 维度 | 项目一：智护云联 | 项目二：多Agent客服系统 |
|------|-----------------|----------------------|
| **类型** | 全栈业务系统 | AI Agent 架构项目 |
| **AI 深度** | 基础 RAG（检索+问答） | ★ 多Agent编排 + Tool Calling + HITL |

### 4.3 核心：State 定义

LangGraph4j 的 `AgentState` 是基于 `Map<String, Object>` 的，通过 **Channels** 定义状态属性的行为。

```java
// CsAgentState.java — 图的自定义状态
public class CsAgentState extends AgentState {

    // Schema 定义：每个字段的 Reducer 行为
    public static final Map<String, Channel<?>> SCHEMA = Map.of(
        // 消息列表：Appender 追加模式
        "messages", AppenderChannel.of(ArrayList::new),
        // Worker 执行结果列表
        "workerResults", AppenderChannel.of(ArrayList::new),
        // Supervisor 下一次路由目标（只保留最新值）
        "next", Channel.of(Channels::lastOf),
        // 轮次计数器（累加）
        "roundCount", Channel.of(
            (a, b) -> (Integer) a + (Integer) b,
            () -> 0  // 默认值
        ),
        // 用户意图（保留最新值）
        "userIntent", Channel.of(Channels::lastOf),
        // 会话ID
        "sessionId", Channel.of(Channels::lastOf)
    );

    public CsAgentState(Map<String, Object> initData) {
        super(initData);
    }

    // ===== 便捷访问方法 =====
    public List<ChatMessage> messages() {
        return value("messages").orElse(List.of());
    }

    @SuppressWarnings("unchecked")
    public List<Map<String, Object>> workerResults() {
        return (List<Map<String, Object>>) value("workerResults").orElse(List.of());
    }

    public String next() {
        return value("next").orElse("FINISH");
    }

    public Integer roundCount() {
        return value("roundCount").orElse(0);
    }

    public String sessionId() {
        return value("sessionId").orElse("");;
    }
}
```

**Channel 类型说明：**

| Channel | 行为 | 适用场景 |
|---------|------|---------|
| `AppenderChannel.of(ArrayList::new)` | 每次更新追加到列表 | messages, workerResults |
| `Channel.of(Channels::lastOf)` | 只保留最后一个值 | next, userIntent, sessionId |
| `Channel.of(reducer, defaultSupplier)` | 自定义 reducer | roundCount (累加) |

### 4.4 核心：Supervisor 节点

```java
// SupervisorNode.java — 大脑节点
@Slf4j
@Component
public class SupervisorNode implements NodeAction<CsAgentState> {

    @Override
    public Map<String, Object> apply(CsAgentState state) {
        log.info("🤖 Supervisor 节点执行，当前轮次：{}", state.roundCount());

        // 安全阀：超过 6 轮强制结束
        if (state.roundCount() >= 6) {
            log.warn("⚠️ 超过最大轮次限制，强制结束");
            return Map.of("next", "FINISH");
        }

        // 构造 LLM 调用
        ChatLanguageModel model = ChatLanguageModel.builder()
            .apiKey("sk-xxx")
            .modelName("deepseek-chat")
            .baseUrl("https://api.deepseek.com")
            .build();

        String prompt = buildSupervisorPrompt(state);
        String response = model.generate(prompt);

        // 解析 Supervisor 决策
        String next = parseDecision(response);
        log.info("🎯 Supervisor 决策: next={}", next);

        return Map.of(
            "next", next,
            "roundCount", state.roundCount() + 1
        );
    }

    private String buildSupervisorPrompt(CsAgentState state) {
        StringBuilder sb = new StringBuilder();
        sb.append("""
            你是一个电商客服系统的调度员（Supervisor）。
            分析用户输入，从以下 Workers 中选择一个处理：

            - order_agent: 订单查询、物流查询、订单修改
            - product_agent: 商品搜索、商品推荐
            - return_agent: 退换货申请、退款计算
            - complaint_agent: 投诉处理、升级人工

            如果问题已经解决，输出: FINISH

            只输出以下格式之一（不要多余内容）：
            {"next": "order_agent"}
            {"next": "product_agent"}
            {"next": "return_agent"}
            {"next": "complaint_agent"}
            {"next": "FINISH"}
            """);

        // 添加上一轮 Worker 的结果（如果有）
        if (!state.workerResults().isEmpty()) {
            sb.append("\n\n【上一步执行结果】\n");
            state.workerResults().forEach(r ->
                sb.append("- ").append(r).append("\n")
            );
            sb.append("\n请根据结果决定下一步。如果 Worker 已经妥善回答，输出 FINISH。");
        }

        return sb.toString();
    }

    private String parseDecision(String response) {
        if (response.contains("FINISH")) return "FINISH";
        if (response.contains("order_agent")) return "order_agent";
        if (response.contains("product_agent")) return "product_agent";
        if (response.contains("return_agent")) return "return_agent";
        if (response.contains("complaint_agent")) return "complaint_agent";
        return "FINISH";  // 兜底
    }
}
```

### 4.5 核心：Worker 节点

Worker 节点使用 LangChain4j 的 Tool Calling，让 LLM 自主决定调哪个 Tool。

```java
// OrderAgentNode.java — 订单 Worker
@Slf4j
@Component
public class OrderAgentNode implements NodeAction<CsAgentState> {

    @Override
    public Map<String, Object> apply(CsAgentState state) {
        String userMessage = extractLastUserMessage(state);
        log.info("📦 OrderWorker 处理: {}", userMessage);

        // 1. 初始化带 Tool 的 LLM
        ChatLanguageModel model = ChatLanguageModel.builder()
            .apiKey("sk-xxx")
            .modelName("deepseek-chat")
            .build();

        // 2. 绑定 Tool
        ToolSpecification orderTool = ToolSpecification.builder()
            .name("queryOrder")
            .description("根据订单号查询订单状态、金额、商品信息")
            .addParameter("orderId", STRING)
            .build();

        ToolSpecification logisticsTool = ToolSpecification.builder()
            .name("queryLogistics")
            .description("查询物流配送进度")
            .addParameter("orderId", STRING)
            .build();

        // 3. 调用 LLM（带 Tool 定义）
        String systemPrompt = """
            你是订单客服专员。根据用户的问题：
            1. 如果需要查订单 → 调用 queryOrder
            2. 如果需要查物流 → 调用 queryLogistics
            3. 回答简洁友好，用中文
            """;

        // LangChain4j 会自动处理 Tool 调用循环
        // （LLM 可能先选择 Tool → 执行 → 再生成最终回答）
        String response = model.generate(
            SystemMessage.from(systemPrompt),
            UserMessage.from(userMessage),
            orderTool, logisticsTool
        );

        // 4. 返回结果
        return Map.of(
            "messages", AiMessage.from(response),
            "workerResults", Map.of(
                "workerName", "order_agent",
                "result", response,
                "success", true
            )
        );
    }

    private String extractLastUserMessage(CsAgentState state) {
        // 从 state 中提取最后一条用户消息
        // 注意：实际 MessagesState 会自动管理消息列表
        return state.messages().stream()
            .filter(m -> m instanceof UserMessage)
            .reduce((a, b) -> b)
            .map(m -> ((UserMessage) m).singleText())
            .orElse("");
    }
}
```

### 4.6 核心：StateGraph 构建

```java
// WorkflowBuilder.java — 构建状态图
@Component
public class WorkflowBuilder {

    @Autowired private SupervisorNode supervisorNode;
    @Autowired private OrderAgentNode orderAgentNode;
    @Autowired private ProductAgentNode productAgentNode;
    @Autowired private ReturnAgentNode returnAgentNode;
    @Autowired private ComplaintAgentNode complaintAgentNode;

    private CompiledGraph<CsAgentState> compiledGraph;

    @PostConstruct
    public void init() {
        this.compiledGraph = buildGraph().compile();
        log.info("✅ LangGraph4j StateGraph 编译完成");
    }

    /**
     * 构建多 Agent StateGraph
     *
     * 图结构：
     * START -> supervisor -> (条件路由) -> Worker
     *                                     -> supervisor (循环)
     *                                     -> FINISH -> END
     */
    public StateGraph<CsAgentState> buildGraph() {
        StateGraph<CsAgentState> graph = new StateGraph<>(
            CsAgentState.SCHEMA,
            CsAgentState::new
        );

        // 1. 注册节点
        graph.addNode("supervisor", supervisorNode);
        graph.addNode("order_agent", orderAgentNode);
        graph.addNode("product_agent", productAgentNode);
        graph.addNode("return_agent", returnAgentNode);
        graph.addNode("complaint_agent", complaintAgentNode);
        graph.addNode("finish", state -> {
            log.info("🏁 流程结束");
            return Map.of();
        });

        // 2. 入口边
        graph.addEdge(START, "supervisor");

        // 3. Supervisor 条件路由
        graph.addConditionalEdges(
            "supervisor",
            edge_async(state -> state.next()),
            Map.of(
                "order_agent", "order_agent",
                "product_agent", "product_agent",
                "return_agent", "return_agent",
                "complaint_agent", "complaint_agent",
                "FINISH", "finish"
            )
        );

        // 4. 所有 Worker 执行完都回到 Supervisor
        graph.addEdge("order_agent", "supervisor");
        graph.addEdge("product_agent", "supervisor");
        graph.addEdge("return_agent", "supervisor");
        graph.addEdge("complaint_agent", "supervisor");

        // 5. 结束边
        graph.addEdge("finish", END);

        return graph;
    }

    public CompiledGraph<CsAgentState> getCompiledGraph() {
        return compiledGraph;
    }
}
```

**关键 API 说明：**

| API | 作用 | 注意事项 |
|-----|------|---------|
| `new StateGraph<>(SCHEMA, constructor)` | 创建状态图 | 第二个参数是 state 工厂方法 |
| `addNode(name, nodeAction)` | 注册节点 | `NodeAction` 同步 / `AsyncNodeAction` 异步 |
| `addEdge(START, "node")` | 设置入口 | 等价于 `setEntryPoint` |
| `addConditionalEdges(node, router, mapping)` | 条件路由 | `router` 返回节点名字符串 |
| `edge_async(state -> ...)` | 创建异步路由 | 用于条件边的 router |
| `compile()` | 编译图为可执行 | 编译后不可变 |

### 4.7 核心：编排入口

```java
// ChatOrchestrator.java — 聊天编排服务
@Slf4j
@Service
public class ChatOrchestrator {

    @Autowired private WorkflowBuilder workflowBuilder;

    // 每个 session 独立的状态
    private final Map<String, CsAgentState> sessions = new ConcurrentHashMap<>();

    /**
     * 处理用户消息（非流式）
     */
    public String processMessage(String sessionId, String userMessage) {
        // 1. 获取或创建会话状态
        CsAgentState state = sessions.computeIfAbsent(sessionId,
            id -> new CsAgentState(new HashMap<>(Map.of(
                "sessionId", id,
                "messages", new ArrayList<ChatMessage>()
            )))
        );

        // 2. 追加用户消息
        state.messages().add(UserMessage.from(userMessage));

        // 3. 执行编译后的图
        try {
            CompiledGraph<CsAgentState> compiled = workflowBuilder.getCompiledGraph();
            var result = compiled.invoke(state);

            // 4. 提取最终回复
            String reply = extractReply(result);
            log.info("💬 [{}] 用户: {} → AI: {}", sessionId, userMessage, reply);
            return reply;

        } catch (Exception e) {
            log.error("Agent 执行出错", e);
            return "抱歉，系统暂时无法处理您的请求，请稍后再试。";
        }
    }

    /**
     * 流式处理（SSE）
     */
    public void processStreaming(String sessionId, String userMessage,
                                  SseEmitter emitter) {
        // 流式版本：使用 StreamingChatLanguageModel
        // 通过 emitter.send(token) 逐片推送
        // 实现方式与非流式类似，但需要异步处理
    }

    private String extractReply(CsAgentState state) {
        return state.messages().stream()
            .filter(m -> m instanceof AiMessage)
            .reduce((a, b) -> b)
            .map(m -> ((AiMessage) m).text())
            .orElse("好的，已为您处理。");
    }
}
```

**为什么用 ConcurrentHashMap 存状态？**
- 比 Redis 简单，展示核心功能
- 面试时可以补充："生产环境我会换成 Redis + CheckpointSaver"

### 4.8 核心：Web 接口

```java
// ChatController.java
@Slf4j
@RestController
@RequestMapping("/api/chat")
public class ChatController {

    @Autowired private ChatOrchestrator orchestrator;

    /**
     * 发送消息（流式 SSE）
     */
    @PostMapping(value = "/send", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public SseEmitter send(@RequestBody ChatRequest request) {
        String sessionId = request.getSessionId() != null
            ? request.getSessionId()
            : UUID.randomUUID().toString();

        SseEmitter emitter = new SseEmitter(0L);  // 0 = 不超时

        // 异步处理，释放 Tomcat 线程
        CompletableFuture.runAsync(() -> {
            try {
                String reply = orchestrator.processMessage(sessionId, request.getMessage());
                emitter.send(SseEmitter.event()
                    .name("message")
                    .data(reply));
                emitter.complete();
            } catch (Exception e) {
                emitter.completeWithError(e);
            }
        });

        return emitter;
    }

    /**
     * 创建新会话
     */
    @PostMapping("/session")
    public Map<String, String> createSession() {
        String sessionId = UUID.randomUUID().toString();
        return Map.of("sessionId", sessionId);
    }

    /**
     * 请求体
     */
    @Data
    public static class ChatRequest {
        private String sessionId;
        @NotEmpty
        private String message;
    }
}
```

## 九、面试可能被追问的问题

| 问题 | 回答方向 |
|------|---------|
| **为什么选 StateGraph 而不是 Chain/Pipeline？** | Chain 是线性无分支。Graph 支持条件路由、循环、并行、HITL，更接近真实场景 |
| **Supervisor 和 Worker 之间怎么通信？** | 通过 AgentState 共享。messages 用 AppenderChannel 追加，所有节点都能读写 |
| **怎么防止 Agent 死循环？** | roundCount 计数器，超过 6 轮强制 FINISH |
| **LLM 返回格式不对怎么办？** | 1) 异常重试 2) 正则兜底 3) 默认走 FINISH |
| **多个用户同时用安全吗？** | 每个 sessionId 独立状态存在 ConcurrentHashMap，互不干扰 |
| **编译图(compile)和调用图(invoke)的区别？** | compile 是静态检查（节点是否连好），invoke 是运行时执行。compile 只需一次 |
| **为什么用 map 存 session 不用 Redis？** | 展示功能用 map 够简单。生产环境我会用 Redis + CheckpointSaver 持久化 |
| **Tool 是 LLM 自动调的吗？** | 是的。LangChain4j 的 @Tool 注解会自动生成 ToolSpecification，LLM 判断调哪个 |
| **如果所有 Agent 都不会回答怎么办？** | 走 fallback："这个问题我转给人工客服了，稍后联系您" |

### 关键的早期里程碑

**Phase 2 是最关键的一步**——跑通一个完整的 Agent 链路：

```
用户发消息 → Controller → Orchestrator
  → StateGraph invoke
    → Supervisor (LLM 决定派给 order_agent)
    → OrderAgent (LLM 调用 queryOrder Tool)
    → Supervisor (决定 FINISH)
    → Finish → END
  → 提取回复 → 返回给用户
```

一旦这个链路跑通了，其他 Worker 就是复制粘贴换 Tools。
