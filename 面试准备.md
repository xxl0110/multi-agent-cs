## 面试 Q&A

### 为什么用 StateGraph 而不是 Chain？

Chain 是线性无分支的。Graph 支持条件路由、循环、并行、Human-in-the-Loop，更接近真实的多 Agent 协作场景。LangGraph4j 的 `StateGraph` 还提供 Channels + Reducers 的细粒度状态管理。

### Supervisor 和 Worker 怎么通信？

通过 `AgentState`（`Map<String, Object>`）共享。`messages` 用 `AppenderChannel` 追加，`next` 用 `base` 只保留最新值，`roundCount` 用自定义 Reducer 累加。

### 怎么防止 Agent 死循环？

`roundCount` 计数器，每轮 +1，超过 6 轮强制 `FINISH`。在 `SupervisorNode.apply()` 中做安全阀判断。

### LangGraph4j 和 LangChain4j 是什么关系？

```
LangChain4j = 每个 Worker 手里的工具（LLM 调用、Tool 执行）
LangGraph4j = 车间主任 + 流水线图纸（节点编排、状态流转）
```

两者互补。项目中两个都用到了：LangGraph4j 画图，LangChain4j 给每个 Worker 提供 LLM 能力和 Tool 调用。

### 这个项目能展示什么？

1. **架构设计能力**：从零搭建 Spring Boot + LangGraph4j 项目，非脚手架
2. **AI 理解深度**：RAG → 多 Agent → Tool Calling → HITL，层次递进
3. **工程规范**：模块划分清晰、异常处理完备、配置化 mock/真实模式
4. **面试区分度**：LangGraph4j 2025 年才发布，比 Spring AI 更前沿

## 容易被追问的技术细节

| 问题 | 回答方向 |
|------|---------|
| **State 为什么用 Map 不用 POJO？** | LangGraph4j 的 `AgentState` 就是基于 `Map<String, Object>` 的，配合 Channels + Reducers 实现字段级更新控制 |
| **编译图(compile)和调用图(invoke)的区别？** | compile 是静态验证（节点连通性、边完整性），invoke 是运行时执行。compile 只需一次，返回线程安全的 `CompiledGraph` |
| **Channels 的几种 Reducer？** | `appender` 追加到列表、`base` + `lastOf` 只保留最新值、`base` + 自定义累加器（如 roundCount） |
| **Agent 节点是怎么注册的？** | 通过 `AsyncNodeAction.node_async(NodeAction)` 包装为异步节点，再用 `graph.addNode(name, action)` 注册 |
| **条件边怎么保证不路由到不存在节点？** | 在 `routeMap` 中定义所有可能的映射，未匹配的会抛出 `GraphStateException` |
| **为什么用 ConcurrentHashMap 存 session？** | 展示核心功能足够简单。生产环境可替换为 Redis + `langgraph4j-redis-saver` |
| **ChatMessage 怎么序列化的？** | 通过 `Map.of("role", role, "content", text)` 格式存储，避免 LangGraph4j 默认 Java 序列化的问题 |
| **Mock 模式和真实模式怎么切换？** | `application.yml` 的 `ai.mock-mode` 开关，运行时通过 `AgentConfig` 条件创建不同 Bean |
| **订单数据是写死的吗？** | 不是。存在 MySQL `cs_order` 表，`OrderTools` 通过 MyBatis Plus 查询，数据可在线增删改 |
